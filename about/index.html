UI Prototyping

Prototypes make everyone happy! Test users and stakeholders are happy because they get an accurate vision of the final product. Designers are happy because they can respond to feedback with minimal clicks and drags. Developers are happy because the prototype communicates precisely what’s to be built.

Building a prototype of an app, website or feature is a great way to solve a lot of design problems early on, before any expensive production code is written. After some intial sketches and dabbling in Photoshop, Illustrator or Sketch, I tend to dive right into the browser and finish the design in HTML/CSS/Javascript.
